                                • A. 참조

                                JAVA 프로젝트에서 MySQL과 같은 RDB와 연동하기 위한 방법은 여러 가지가 있다. 대표적으로 MyBatis, JPA 등이 있는데 본 포스트에서는 자바 스프링부트 프로젝트에서 MySQL과 연동하기 위해 JPA를 활용한 개발을 다룰 것이다.

                                # JPA란?

                                JPA는 Java Persistence API의 약자로 Java ORM(Object Relation Mapping, 객체 관계 매핑) 기술에 대한 API 표준 명세를 말한다. 이는 자바 어플리케이션에서 RDB를 사용하는 방식을 정의한 인터페이스다. JPA는 단순한 명세이기 때문에 JPA만 가지고는 어떤 구현 기술을 사용하지 못한다. 실제로 우리가 사용하는 구현 기술은 Spring Data JPA로부터 제공되는 것이다. Spring Data JPA는 JPA를 간편하게 사용하도록 만든 오픈소스 라이브러리다. Spring Data JPA는 개발자가 JPA를 한 단계 더 추상화시킨 Repository 인터페이스만을 사용하게 하고 내부적으로는 JPA 인터페이스 EntityManagerFactory, EntityManager, EntityTransaction 등을 상속받은 JPA 구현체들을 이용해 기능을 동작시킨다. JPA 구현체에는 Hibernate, DataNucleus, EclipseLink 등이 있다.

                                # MySQL 환경 구성

                                본 블로그의 MySQL 설치open in new window 포스트를 참고하자. 혹은 강좌에서 가이드한 대로 MySQL 공식 홈페이지에서 설치 프로그램을 실행하여 설치하자.

                                # application.properties 설정

                                다음과 같이 MySQL DB와 JPA와 관련된 설정을 명시한다.

                                프로젝트가 application.properties를 사용할 경우

                                src/main/resources/application.properties
                                # MySQL 드라이버 설정
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# DB URL
spring.datasource.url=jdbc:mysql://<DB_IP>:<DB_PORT>/<DB_NAME>?useSSL=false&useUnicode=true&serverTimezone=Asia/Seoul 

# DB username
spring.datasource.username=<DB_USER_NAME>

# DB password
spring.datasource.password=<DB_PASSWORD>

# JPA쿼리문 확인여부
spring.jpa.show-sql=true

# DDL(create, alter, drop) 정의시 DB의 고유 기능을 사용할 수 있다.
spring.jpa.hibernate.ddl-auto=update

# JPA의 구현체인 Hibernate가 동작하면서 발생한 SQL의 가독성을 높여준다.
spring.jpa.properties.hibernate.format_sql=true

                                프로젝트가 application.yml을 사용할 경우

                                src/main/resources/application.yml
                                spring:
  jpa:
    hibernate:
      ddl-auto: update
    properties:
      hibernate:
        show_sql: true
        format_sql: true
