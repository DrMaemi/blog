                        • A. 참조

                        본문은 2019년도 하반기 아주대학교의 컴퓨터공학과 교수로 재직하셨던 강경란 교수님의 컴퓨터네트워크 과목 강의 자료를 기반으로 작성되었음을 알립니다.

                        # 1. 네트워크(Network)란 무엇인가?

                        네트워크란 어떤 형태로든 상호 연결하여 나타낼 수 있는 Group 또는 System을 말한다.

                        E.g., Biological network, Business network, Computer network, Social network, ...

                        그 중에서 컴퓨터 네트워크란 자원을 공유하거나 전기적 통신을 하기 위해 2개 이상의 컴퓨터가 서로 연결된 망을 뜻한다.

                        # 2. 인터넷(Internet)이란 무엇인가?

                        # 2.1. 서비스 관점

                        인터넷은 다음 성질을 만족하는 인프라(Infrastructure)다.

                        • 어플리케이션에 서비스를 제공함
                          • Web, VoIP, email, games, e-commerce, social nets, ...
                        • 어플리케이션에 프로그래밍 인터페이스를 제공함
                          • 어플리케이션 프로그램이 인터넷에 접속할 수 있도록

                        # 2.2. "Nuts and Bolts" 관점

                        인터넷은 다음과 같은 것들을 포함하는 거대한 집합체다.

                        • 수백만개의 컴퓨팅 장치
                          • Hosts = end systems
                        • 데이터 통로, Communication links
                          • E.g., Fiber, copper, radio, satelite
                          • E.g., Transmission rate: bandwidth
                        • 패킷 스위치(Packet switches)
                          • 라우터와 스위치들을 통한 패킷 전송
                        • 네트워크들의 네트워크(Network of networks)
                          • ISP간 상호 연결(Interconnected ISPs)
                        • 프로토콜(Protocols)
                          • 인터넷을 구성하는 SW 컴포넌트
                          • E.g., TCP, IP, HTTP, Skype, 802.11
                        • 인터넷 표준(Internet standards)
                          • 철저하게 테스트된 명세(Specification), 규정된 형식(Formalized regulation)
                          • 인터넷 통신 시 인터넷 표준을 반드시 따라야 함
                          • E.g., RFC, IETF

                        # 3. 프로토콜(Protocol)이란 무엇인가?

                        프로토콜(Protocol)이란 논리적/물리적으로 구분된 2개의 객체(Entity)가 서로 성공적으로 통신하기 위해 반드시 지켜야 하는 규약들의 집합(The set of conventions)이다.

                        프로토콜은 무엇을(What), 어떻게(How), 언제(When) 통신하는가를 설명한다.

                        인터넷 상의 모든 통신 활동은 정해진 프로토콜을 따라야 하며, 프로토콜은 네트워크에 존재하는 객체들이 송수신하는 메세지(데이터)의 형식, 순서와 송수신 방법을 정의한다.

                        # 4. 네트워크 구조 개요

                        <그림 2. 네트워크 구조 개요>

                        네트워크를 구성하는 요소는 크게 세 가지로 나누어 볼 수 있다.

                        • Network Edge
                          • Hosts: 클라이언트와 서버들
                          • 서버는 보통 데이터 센터에 있음
                        • Access networks & Physical media
                          • 네트워크에 접속하기 위해, 통신에 사용되는 물리적 매체
                          • Wired, wireless communication links
                        • Network core
                          • 상호 연결된 라우터들과 ISPs

                        인터넷 망에 접속하기 위한 단말 네트워크를 환경에 따라 Home network(Residential access nets), Mobile network(Wireless access network), Institutional network(Enterprise access nets)로 분류할 수 있다. Institutional network는 학교나 기업 환경의 네트워크를 뜻한다.

                        # 4.2. Institutional network

                        <그림 4. Institutional network 개요>

                        • 1Gbps, 10Gbps 이상의 전송 속도(transmission rate)을 가지는 고속 인터넷 망을 사용한다.
                        • 오늘날 단말 시스템(End system)은 보통 Ethernet switch에 연결되어 인터넷 망에 접속한다.

                        # 4.3. Mobile network

                        단말 시스템은 기지국(Base station) 또는 공유기(Access Point)를 통한 공용 무선 네트워크 접속(Shared wireless access network) 방식으로 인터넷 망에 접속한다.
